# Agent 三件套修改须前置协商

## 事件

2026-04-24 ~ 2026-04-25，architect 升格 MKO（Module Knowledge Open-Closed）准则到 soul.md 和 identity.md，落盘后才通过 channel review 通知人事管理审计。内容本身审计通过，但流程属于"先斩后奏"。

## 根因

soul.md / identity.md / agent.json 属于 agent 长期 DNA，修改影响 agent 跨会话行为。无前置知会机制时，人事管理只能事后审计——若内容有问题，已落盘更难撤销，且可能已影响其他会话。

## 准则

**soul.md / identity.md / agent.json 任何段落级修改（新增段落 / 替换信念 / 删除准则）**，执行 agent 须：

1. 先在 channel 发 `message` 告知人事管理，说明：变更内容、变更理由、计划落点
2. 等人事管理回复确认（或 24h 无异议视为通过）
3. 落盘后走正式 channel review 做事后归档

**豁免**：拼写修正、格式对齐等不影响语义的轻量改动不需要前置协商。

## 对冲动作

- architect identity.md § 注意 已补入此约束（2026-04-25，用户确认）
- 其他 agent 若有类似修改需求，同等约束适用

## 普遍性

此规则适用于所有已注册 agent（architect / director / coder / manager 自身），凡涉及三件套段落级修改均须前置协商。
