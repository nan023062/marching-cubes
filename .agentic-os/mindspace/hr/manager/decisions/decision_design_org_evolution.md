# 设计组织演化路线：当前全能 → 未来分层

## 当前阶段（现在）

`designer` 一个 agent 负责**所有设计职责**，包括但不限于：
- 产品设计（产品目标对齐、功能边界判断、竞品研究）
- UX 设计（信息架构、交互流程、视觉规范、原型）
- 其他设计职责（视未来需要扩展）

**当前 agent 定位**：全能设计师，产物覆盖全设计栈。`name: designer`，`title: 产品设计师`。

## 演化方向（未来）

通过多轮 session 沉淀，当设计工作量增加、边界清晰时，按以下结构拆分：

```
总设计师（Chief Designer）
  ├── 产品设计师（Product Designer）
  ├── UX 设计师（UX Designer）
  └── xx 设计师（按需新增：视觉/运营/品牌等）
```

**拆分触发条件**（人事管理判断，用户确认）：
- designer 中期记忆中某一设计类别的条目量超过其他类别 2 倍以上（专业化信号）
- 某设计方向出现稳定的独立 skill 候选（3+ skill）
- 单轮 session 工作量超过合理范围，designer 无法在质量标准下全覆盖

## 对当前 designer 的影响

1. **技能发展要有宽度** — 不能只深耕 UX 交互，产品思维、竞品研究、视觉规范都要有覆盖。未来拆分时，当前 designer 的技能库是分拆的原材料。
2. **identity 写法要便于拆分** — 职责描述按"产品设计 / UX 设计 / 其他"分块，不混写，方便未来抽离。
3. **skill 打标** — designer 沉淀的 skill 要在 frontmatter 里标清楚 `domain: product | ux | visual | ...`，便于未来按 domain 划入对应子 agent。

## 总设计师角色预判

未来拆分后，当前 `designer` agent 最可能演化为**总设计师**（协调多个专业 designer），而非下沉为某一专科。总设计师的核心能力：
- 跨设计域决策（取舍优先级）
- 设计系统治理（所有专科共用的 token/组件）
- 产品目标对齐（向用户/PM 反向）
- 专科设计师的工作验收

## 不决策的内容

- 具体拆分时间点：由沉淀自然触发，不人为规划
- 拆分后 agent 的三件套：到时候按实际积累的 session + 中期记忆起草，不现在预设
